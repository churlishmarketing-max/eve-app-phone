## Operating rules (binding)

1. **Interview before assuming.** If a requirement is ambiguous, ask Brandon —
   one question at a time. If he's unavailable and the assumption is cheap to
   reverse, build it and flag the assumption at the top of your report.
2. **⚑VERIFY convention.** Anywhere this package marks ⚑VERIFY, the detail
   (an SDK method name, a plugin, a config step) must be checked against live
   official docs before use — these things change. Start with
   https://docs.claude.com/en/docs_site_map.md for Agent SDK / API questions.
3. **Never fake done.** Every phase has a Definition of Done below. Ship each
   phase with a verification receipt: the DoD item, the evidence, the command
   or screenshot that proves it.
4. **No canned personality strings.** Every EVE message the user sees is
   generated by the brain in character. Hardcoded reply text anywhere outside
   the demo shell is a spec violation (see quality bar, `01`, §6).
5. **Secrets in env only.** Nothing sensitive in the repo, the APK, or a
   screenshot. `.gitignore` before first commit.
6. **Autonomy tiers are enforced in code, not vibes.** Any tool that sends
   something external (email, SMS, Slack, publish) requires an explicit
   confirmation round-trip with Brandon. No exceptions, ever (see `02` §6).

## Build phases and Definitions of Done

### Phase 1 — HER VOICE (target: days 1–3)
Brain service running with the Character Bible + doctrine layers; React app
ported from the demo; streaming text chat; Capacitor APK sideloaded; FCM wired;
7:00 AM morning brief lands on the lock screen.

**DoD:**
- [ ] `POST /chat` streams an in-character reply end-to-end from the phone
- [ ] Conversation quality bar (`01` §6) passes on the 6-prompt tone suite
- [ ] APK installed on Brandon's Android via sideload; survives app kill/restart
- [ ] A scheduled morning brief push arrives with live ledger content, ≤25 words
- [ ] No secrets in the repo or APK

### Phase 2 — HER MEMORY (target: days 4–7)
Full Supabase schema; context assembly on every message; nightly distillation;
recall tool; client pulse tables and sweep; Today screen reads live data.

**DoD:**
- [ ] "What did we decide about X yesterday?" answered correctly from memory
- [ ] Distillation job produced memory entries from a real day of chat
- [ ] A client with `last_touch` past cadence generates an attention item + push
- [ ] She declines to invent a memory she doesn't have (test with a fake premise)

### Phase 3 — HER HANDS (target: week 2)
Connectors via MCP (Gmail read/draft, Calendar read/write, Notion, Slack read);
fleet dispatch (`/dispatch` + jobs table + Ops screen); voice pipeline
(Deepgram in, ElevenLabs out).

**DoD:**
- [ ] "Put X on my calendar Thursday at 2" creates the real event
- [ ] "Summarize my unread email" works; drafted replies never auto-send
- [ ] A dispatched job (e.g. proposal draft) runs, lands a deliverable, pings done
- [ ] Full voice loop: speak → transcript → reply audio in her voice, streaming

### Phase 4 — HER SENSES (target: week 3)
Android notification listener + SMS read; dictated SMS reply with confirm;
wardrobe pipeline (Higgsfield generate → approve → wear; Soul training once a
founding look is chosen); routines with streaks.

**DoD:**
- [ ] Incoming text is read/summarized on request; reply sends only on confirm
- [ ] Wardrobe: generate → approve → wear round-trip with real renders
- [ ] A routine streak increments only same-day (no back-dating — by design)

### Phase 5 — HER REACH (later, separate session)
Update the Even Realities G2 plugin to hit the new brain — it inherits memory
and personality for free. Not in scope for this package beyond the API being
glasses-friendly (`02` §3).
