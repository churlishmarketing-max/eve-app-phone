# 04 — PROACTIVE ENGINE

This is the half of EVE that makes her an assistant rather than a chatbot: she
reaches out first. All schedules fire from n8n (crons hitting `POST /job`);
all message **text is generated by the brain in character** at send time —
n8n carries triggers, never copy.

## 1. The daily cadence (defaults; Brandon can tune each)

| Time | Job | Content |
|---|---|---|
| 7:00 | `morning_brief` | Today's Three, calendar, the one avoided thing, floor status. ≤25 words in the push; full brief on the Today screen. |
| 11:45 (weekdays) | `floor_check` | Sales-conversation floor progress vs 3/week; only pushes if behind pace. |
| 12:30 | `pulse_sweep` | Client pulse digest (see §3) — pushes only when something's quiet. |
| event-driven | `tripwire` | Ad frequency ≥3.0, funnel breaks, deadline collisions. The only alerts styled red. |
| 17:30 | `closeout` | Shipped vs slipped; slipped items auto-rescheduled; the go-be-a-person line. |
| 02:00 | `distill` | Nightly memory distillation (`03` §5). No push. |
| Sunday 19:00 (optional) | `week_preview` | Next week's shape; sales blocks placed first. |

**Quiet hours: 21:30–06:30 and all of Brandon's family-blocked calendar time.**
Nothing pushes during quiet hours by default; genuinely urgent tripwires queue
for the morning brief unless Brandon later defines an emergency class.

## 2. Attention rules (the engine)

Evaluated by `pulse_sweep` and event hooks; each hit creates/updates an
`attention_items` row:

| Rule | Trigger |
|---|---|
| `silent_client` | `now - clients.last_touch_at > cadence_days` |
| `review_due` | per-client review cadence reached (store on `clients` when added) |
| `overdue` | task past due; nudge_level escalates per §4 |
| `renewal` | retainer enters window in ≤21 days (Guardian's lane) |
| `routine_risk` | daily routine unticked by 20:00 |
| `stale_domain` | a life/work domain with zero activity in N days (later, P4) |

## 3. Client pulse — the touch-base radar

The feature Brandon asked for by name: never let a client sit in silence.

Sweep algorithm:
```
for client in clients where status='active':
  days_quiet = now - last_touch_at
  if days_quiet > cadence_days and no unresolved silent_client item:
      context = last touches summary + open jobs/deliverables for client
      draft   = brain writes the touch-base update (what to say and why)
      create attention_item(kind=silent_client, message, ref={client, draft})
      push (compressed register)
```
The item always carries **what to touch base about** — the last deliverable,
the unreacted ship, the number worth sharing — plus the drafted update. Sending
is RED: she drafts, Brandon sends (or approves send in Phase 3+).

## 4. Nudge escalation law — usefulness, never volume

`attention_items.nudge_level` governs reminder behavior:

- **N1 — inform.** State it once, cleanly.
- **N2 — shrink the task.** The reminder arrives with the work made smaller:
  draft attached, email trimmed to three sentences, calendar hold placed.
- **N3 — thumb-only.** Everything is done except the confirmation. "All that's
  left of the job is your thumb."
- There is no N4-louder. If N3 sits for 48h, it goes in the close-out and the
  Friday report as *slipped*, once, without nagging.

This law is the codified fix for v1's nagging problem. It binds every reminder
in the system.

## 5. Push payloads (FCM)

Three notification channels so Android lets Brandon tune them independently:
`brief`, `nudge`, `tripwire` (⚑VERIFY Capacitor push plugin channel setup).

```json
{
  "notification": { "title": "EVE · CLIENT PULSE", "body": "<generated, ≤25 words>" },
  "data": { "kind": "silent_client", "attention_id": "<uuid>", "deeplink": "eve://ops" }
}
```

Compression register from the Character Bible §10 applies: substance first,
exactly one clause of flavor. Deep links open the relevant tab/card.

## 6. Capture inbound (completes the loop)

- **App:** `POST /capture` with text or a voice-note transcript.
- **Email:** dedicated address → n8n webhook → `/capture`. Subject containing
  "new task" (case-insensitive) forces task creation; the email is summarized,
  filed to the matched client, `source_link` preserved for one-tap return to
  the thread.
- Anything unmatchable lands in the inbox with an attention item — the inbox
  exists to be emptied.
